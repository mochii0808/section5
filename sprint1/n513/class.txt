객체 지향 프로그래밍, OOP
- 모든 실체를 클래스, 인스턴스, 함수, 변수로 객체화하여 프로그래밍
- !!기능 재사용 가능하도록 설계!!

객체
- 속성과 기능을 가진 것
- 클래스 / 인스턴스

클래스
- 설계도
- 특징에 따른 양식 설정

인스턴스
- 실체
- 관계 설명 위주
ex) a = Cookie()일 때
    - a는 객체
    - a는 Cookie의 인스턴스

-------------------------------------------------------------------------

캡슐화
- 내부 속성과 함수를 하나로 묶어 클래스로 선언하는 개념

상속(Inheritance)
- 상위 클래스의 모든 기능 재사용 가능
- __init__ 불필요

포함(Composition)
- 다른 클래스의 일부 함수만 재사용
- __init__ 필요
